Joining along time axis¶

One may find in need to join to files in time axis since the typical individual file coming from callisto is of 15 minutes and an event or feature may extend more than that.
In this case jointimeaxis function is useful. To use this you need to provide following inputs

  1. the name/path of first input fits file
  2. the name/path of second input fits file
  3. the name/path of final stiched file to be saved.

These input files are supposed to have same dimentions in frequency and should be continuous in time (though a gap of 1 sec is ignored) and should have the same time-sampling these things are checked and if not followed function won't go ahead with joining operation. If your inputs files are from the same observatory you should not bother about these they should match naturally(unless, if you have changed them)

Let us join these two fits files and save on disk.

In [6]:
pc.jointimeaxis('IISERP_20151104_031152_59.fit', 'IISERP_20151104_032652_59.fit', 'joined.fits')

Let us plot the joined.fits file.

In [7]:
pc.plot('joined.fits', 'joined.png', option=3, blevel=127)

Now, here in this plot one can see a feature, nice in the middle, which was on the edge of the first input fits image.

In this way, ideally, one can join all the data of whole day.

If you further feel a need to slice only the central part of the joined fits image on time axis putting the feature of interest in right middle, one can do so as shown in the slicing example with proper time inputs.
